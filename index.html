Writing R Functions with the Diamonds Dataset

1. Basic Functions

# Calculate price per carat
price_ratio <- function(price, carat) {
  price / carat
}

diamonds |> mutate(value = price_ratio(price, carat))

Data Ethics Consideration

The diamonds dataset shows that Ideal-cut diamonds average $3,458 while Fair-cut average $4,359 despite lower quality.

Discussion: How might a function that recommends "best value" diamonds based solely on price/carat perpetuate market biases against certain cuts?

2. Data Frame Functions

# Flag expensive diamonds
flag_expensive <- function(df, threshold) {
  df |> mutate(expensive = price > threshold)
}

diamonds |> flag_expensive(5000)

Data Ethics Consideration

In the data, D-color (most rare) diamonds are 3x more expensive than J-color, but this doesn't reflect actual visibility differences to the naked eye.

Discussion: Should a function that flags "overpriced" diamonds account for subjective human perception of color grades?

3. Plot Functions

# Plot price distribution by clarity
plot_clarity <- function(df) {
  ggplot(df, aes(x=clarity, y=price)) + 
    geom_boxplot() +
    scale_y_log10()
}

Data Ethics Consideration

The plot shows SI1 clarity diamonds (most common) appear "low quality" due to price compression, while ultra-rare IF grades dominate the high end.

Discussion: How might using a log-scale in visualizations mask affordability barriers for average consumers?
